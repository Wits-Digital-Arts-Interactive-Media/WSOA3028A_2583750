The mapping is done on the wireframes and displays the chronological order in which users view the webpage. In this map, it displays the left-right bias, which is a cognitive effect where people tend to process information and scan visual elements from left to right. For user interface we focus on placing important information towards the left of the screen, however positional memory also plays a key role as the user is likely to remember according to placement, as headings that are either first or last are remembered more.

Users should be able to navigate the website with ease and view what they want with no difficulty. Users should be able to view my portfolio and profile, academic essays, UI design capabilities, and blog posts about various topics regarding technology and development.

Style Guide and Research

Hicks Law:
Hicks Law states that the time it takes for a user to make a choice increases with the number and complexity of choices available to them. Adhering to this law, I wanted the interface for my website to have minimal choices involved. Fewer options allows for simple and quick navigation.
Positional Memory:
Chances are users have already visited hundreds, maybe thousands of websites before and therefore there are certain conventions that most websites adhere to. Example: the navigational bar will be near the top of the homepage so that users can decide where on the site they would like to visit.
